Require References

There are two main functions of the require reference.

  1. Load a referenced file and parse any sub references
  2. Same as #1, but export the required files to a variable

Load a referenced file

If you want to reference a child file and load all of its required references, use one of the following options. 

    // The standard require reference path
    //= require("dialog.js");
    // or
    //= require('dialog.js');

    // Referencing a require without quotes
    //= require dialog.js

    // Referencing a require as an element for resharper support
    /// <reference path="dialog.js" />

    // Referencing a require file under a folder
    //= require("helpers/dialog.js");

Load a reference variable file

If you want to reference a child file and load the referenced file into a variable, use one of the following options. 

    // Require a file and use it as a variable on the page
    var popup = require("plugins/popup.js");

    // Require a file and use it for an object on the page
    var helpers = {
        validations: require("contants.js").validations,
        message: require("contants.js").messages
    };

When you use a variable require reference, the referenced child file needs to specify what is being exported. This is a pattern found in CommonJS and Node.js. To do this, use module.exports at the bottom of your referenced script. In the example below, the isMinLength method is the only object exported. The variable length stays private to the file. 

    // validations.js file

    var length = 10;

    var isMinLength = function(text) {
        if (!text)
            return false;

        if (text.length < length) 
            return false;

        return true;
    };

    module.exports = minLength;

    // app.js file

    var minLength = require("validations.js");
    if (!minLength("something"))
        alert("The text isn't the minimum required length");
