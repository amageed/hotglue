Plug-ins

Through installing plug-ins, you are able to extend the base functionality of Hotglue to meet your needs. There are five different types of plug-ins available to install. For additional information on the plug-ins, and specific configuration information click the learn more below each section for additional information.

Frameworks

Hotglue's only required component is a framework. It's through a framework that Hotglue is executed and prepares the scripts. The first thing you need to install to get Hotglue to run is the framework. Once the framework is installed, any of the remaining plug-ins can be installed in any order.

Learn more about specific frameworks

Compressors

By default, Hotglue loads all of your files as is. With the debug option set to false, Hotglue will bundle all the referenced files into one, but the bundled file will still be uncompressed. If you want to add additional compression, you can install one of the compressor plug-ins to minimize file size. The compressor only runs when the debug option is set to false.

Learn more about compressor options

Script References

To expand the default way Hotglue loads the required reference files, you can install a script reference plug-in. These plug-ins don't expand any base functionality, they are focused solely on changing how files are loaded from the HTML page.

Learn more about script reference options

Compilers

Hotglue provides the ability to expand the languages available to program in. Through the compiler plug-ins, Hotglue will build and compile the alternate language files on the fly. Once a file has been compiled, Hotglue will cache the file to serve it up faster, until it is changed again.

Learn more about compiler options

Templates

To abstract any template files from your HTML and from your scripts, Hotglue provides template plug-ins to help simplify the use and storage of these template engines. Every template works the same, in that all you need to do is execute the required template variable. To execute the template, call the variable as a method and pass in any template variables. The template will have already been configured, and will return a string generated from the template engine. 

    var template = require("template.tmpl");
    var data = { name: "Bob", age: 25 };
    var html = template(data);

Learn more about template options
