Page References Details

In order to get script references onto your page, you need to reference a script file through the Hotglue library. When you reference a script file, below is a detailed explanation of what happens behind the scenes on a page load.

  1. Find and parse all Hotglue library and require references
  2. Build a list of all required files, by going through the reference dependency chain
  3. Remove any duplicate library and require references
  4. Ensure there aren't any circular references
  5. Order references to load from no required child references, to no parent references
  6. Generate all script references and write them to the page

After this process has happened for the first time, the found references and any compiled scripts are cached. On subsequent page loads, the cached versions are served up. If any of the files have changed since they were last cached, the process starts again listed above.

Single Page Reference

Hotglue works best, at the moment, when there is a single page reference. This is because of how Hotglue processes each referenced file. If you added more than one js reference files at the moment, the process listed above would be repeated for each referenced file. This could lead to duplicate scripts since each referenced file is processed separately. The best solution is to put all required references under a single script file and handle everything within that file. 

    // Asp.net Page with two references could lead to duplicate script references
    // A better solution is to include a require reference to dialog.js in app.js
    <%= HotGlue.Script.Reference("Accounts/app.js") %>
    <%= HotGlue.Script.Reference("Accounts/dialog.js") %>
