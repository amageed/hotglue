Hopefully from the quick start documentation you were able to see that getting Hotglue up and running should be fairly easy. Now lets go over some of the options you have available to use with Hotglue.

File Dependency Graphing

One of the main features of Hotglue, is finding and loading all required files dynamically through file references. There are two kinds of references available in Hotglue to you in your scripts, they are Library, and Require. The way Hotglue knows how to initially load this references is through the page reference.

Page Reference

Each page should have one main file to load. This file then has additional dependencies to load, but as far as your page reference is concerned, there is only one point of entry. This can be named what ever you would like, but it is assumed that the file you are loading is referenced under your /Scripts folder in your web project.

Everything in Hotglue is assumed to be a relative path unless it starts with / or ~/. If you wanted to load app.js and it was in the path Scripts/Accounts/app.js, it would be referenced like this: 

    // Asp.net Page
    <%= HotGlue.Script.Reference("Accounts/app.js") %>

As far as referencing a script in your page, this is all you would need to do. Hotglue will traverse and find all the required references for you automatically.

Learn more about page references

Library Script References

The first method to load scripts is through the Library method. When loading other scripts with the library method, the scripts are loaded as is and not checked for additional references. It is recommended that you load all third-party scripts with this method.

The patterns for loading a reference are pretty flexible. The default reference is below. 

    //= library("jquery.js");

Learn more about library references

Require Script References

The second method to load scripts is through the Require method. This method is where the real power of Hotglue comes into play. There are three different ways to use require references. 

    // Require a file and all of its sub files
    //= require("includes.js");

    // Require a file and use it as a variable on the page
    var popup = require("plugins/popup.js");

    // Require a file and use it for an object on the page
    var helpers = {
        validations: require("contants.js").validations,
        message: require("contants.js").messages
    };

If you use the variable reference method you need to treat that file as a module and export what you need from it. For specifics on this, view more on required references.

Learn more about require references

File Bundling

Hotglue has built-in bundling to combine all of the referenced Hotglue files into one. The bundling works automatically based on the Debug or Release setting in your project. For an Asp.net project, this means the Web.config attribute is set to debug="false" to bundle all of the files. When debug is set to true, all of referenced script files will load individually.
