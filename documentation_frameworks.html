Frameworks

Default

There is no default framework for Hotglue, you must install one of the plug-ins below that correspond to the project type you are working on. After a framework is installed, you can add additional functionality through additional plug-ins.

All framework plug-ins provide the following base functionality:

  • Parse JavaScript files for dependent file references
  • Cache previously parsed files for faster loading
  • Bundle files together into a single file on release mode
  • Ability to have require reference module files

Asp.net MVC and Web forms

To get Hotglue up and running on Asp.net, all you need to do is install the Nuget package and start using it. 

Install-Package HotGlue.Aspnet

Nancy

There are two ways to install Hotglue for Nancy. If you are using the Razor view engine install the Razor package. This allows you to reference the same way you do in Asp.Net. 

Install-Package HotGlue.Nancy.Razor

If you are using a different view engine you can install the default Nancy package. For the Super Simple View Engine you'll need to pass in your reference as a variable. Other view engines have not been tested. 

Install-Package HotGlue.Nancy
