Compilers

Default

By default, Hotglue does not support any languages other than JavaScript. To expand Hotglues supported languages, you can install a compiler plug-in. Compilers look for specific file extensions to know when to process a file.

CoffeeScript

To use CoffeeScript with Hotglue, you will need to install the plug-in and name the CoffeeScript files with the extension .coffee. As far as using Hotglue references, it is a straight cross over from the JavaScript references. 

    # Library reference
    #= library("jQuery.js")

    # The standard require reference path
    #= require("dialog.js")
    # or
    #= require('dialog.js')

    # Referencing a require without quotes
    #= require dialog.js

    # Require a file and use it as a variable on the page
    popup = require("plugins/popup.js")

    # Require a file and use it for an object on the page
    helpers = {
        validations: require("contants.js").validations,
        message: require("contants.js").messages
    }

TypeScript

Since TypeScript is a superset of JavaScript, all of the Hotglue references work the exact same as they do for JavaScript. The only difference is TypeScript files end with the extension of .ts rather than .js.
